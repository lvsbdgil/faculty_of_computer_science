# faculty_of_computer_science

### Описание мини-проекта:
## *"Вам дана конфигурация частично заполненного поля игры крестики-нолики. Необходимо, чтобы Ваше приложение сделало следующий наиболее разумный ход."*

## Уточнение формулировки:
* **Человек играет за нолики, а алгоритм за крестики**
* **Входные данные - стандартное поле для игры в крестики-нолики(3x3), где следующий ход должен выполняться крестиками, то есть количество крестиков и ноликов равно**
* **Выходные данные - поле формата 3x3, где максимально рационально расположен новый крестик**

## Цели:
* **Сделать удобный ввод конфигураций поля для пользователя**
* **Определить оптимальный алгоритм для определние наиболее разумного хода** 
* **Реализовать вывод получившегося поля для пользователя**

## Реализация:
* **Буду использовать алгоритм минимакс, так как с помощью него можно определить максимально рациональный ход для компьютера, который в то-же время будет максимально неудобен для игрока** 
* **Пользователь будет вводить в консоль поле в таком формате:** \
X-X\
00-\
0-X

* **Алгоритм будет выводить получившееся поле в консоль в формате:** \
X - X\
0 0 X\
0 - X

***Примечание крестик - это заглавная буква икс на английской раскладке, нолик это ноль, пустая клетка это короткое тире***
